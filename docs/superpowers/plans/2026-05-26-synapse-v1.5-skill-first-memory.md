# Synapse v1.5 Skill-First Memory Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Turn Synapse into a stable, lightweight Claude Code personal engineering memory skill.

**Architecture:** Keep Markdown nodes as the source of truth, keep bash scripts as user-facing entry points, and move fragile or expensive logic into focused Python helpers. The v1.5 scope stabilizes the existing skill workflow before adding heavier code-awareness features.

**Tech Stack:** Claude Code Skills, bash 4 wrappers, Python stdlib, Markdown memory nodes, SQLite as derived cache only.

---

## File Structure

- `docs/superpowers/plans/2026-05-26-synapse-v1.5-skill-first-memory.md`: Tracks this implementation plan.
- `tests/test_runner.sh`: Structural skill validation entry point.
- `tests/test_suggest_edges_dedup.sh`: Regression test for noisy duplicate edge suggestions.
- `synapse-graph-memory/scripts/suggest_edges.sh`: Edge suggestion logic and project path parsing.
- `synapse-graph-memory/scripts/generate_memory_map.py`: New Python MAP engine in later tasks.
- `synapse-graph-memory/scripts/generate_memory_map.sh`: Thin compatibility wrapper after the Python engine is introduced.
- `synapse-graph-memory/scripts/synapse_note.sh`: Manual and automatic memory capture pipeline.
- `synapse-graph-memory/examples/solo-saas/`: Versioned fixture for release checks and smoke tests.
